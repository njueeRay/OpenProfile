## 2. 各角色审阅发现

### 2.1 Brain 的发现

**项目特定泄露（5 处）：**

| 位置 | 泄露内容 | 建议处理 |
|------|---------|---------|
| §1.4 | "复盘会议 #003 教训" | 去掉会议编号引用，保留原则本身 |
| §4.3 | 常用 Scope 列表中包含 `astro`、`blog` | 改为通用示例，项目特定 scope 说明"由 Brain 在接手新项目时写入 copilot-instructions.md" |
| §4.4 | 示例中写死了 `njueeRay-profile, OpenProfile` | 改为 `{repo-a}, {repo-b}` 占位符示例 |
| §7 引言 | "来自 code-reviewer 的铁律（复盘会议 #003）" | 去掉会议编号 |
| §11 | "OpenProfile 方法论包" | 改为"团队方法论包" |

**体系性缺失（2 处）：**

1. **缺少 Playbook 定制指南** — Brain 带团队进入新项目时，如何 fork 和定制本手册？当前 §9 Pre-flight 只覆盖了仓库配置，没有覆盖 Playbook 本身的适配流程
2. **缺少"零上下文冷启动"协议** — §12 假设项目已有 copilot-instructions.md 等文件。但如果进入一个全新的、空白仓库呢？Brain 需要一套从零开始建立全部上下文的协议

### 2.2 PM 的发现

**冗余（3 处）：**

| 冗余对 | 重叠内容 | 建议 |
|--------|---------|------|
| §5.2 + §15.4 | 两处都描述了 Release 操作步骤（tag → push → create release） | §15.4 只保留 API 特有部分，其他步骤引用 §5.2 |
| §4.2 + §4.5 | Emoji 表把 Type 含义重新列了一遍 | §4.5 只保留 emoji↔type 映射，去掉含义列，引用 §4.2 |
| §2.2 + §10 | 会话关闭协议第 1 步就是"逐项核查 DoD"，而 §10 又完整列了 DoD | 这是合理引用，不算冗余。保留。 |

**版本号建议：**

> [Unreleased] 已积累 30+ 条变更，涵盖 §4.5/§5.4/§12/§13/§14/§15/§16 + 本次的项目无关化重构。
> 这不是 PATCH，也不只是 MINOR。
>
> 这是一次**根本定位转变**：从"OpenProfile 项目手册"变成"可移植的团队方法论"。
>
> **建议：v3.0.0**

### 2.3 Dev 的发现

**跨平台问题（1 处）：**

| 位置 | 问题 | 建议 |
|------|------|------|
| §15.2 | Token 获取只有 PowerShell 脚本，macOS/Linux 团队无法使用 | 补充 bash 等效命令 |

**§9.2 路径硬编码：**

> 仓库基础配置中列了 `docs/` 子文件夹名：`design-decisions / component-guide / agent-workflow`。
> 这些是 OpenProfile 的命名惯例。建议改为：
> "Brain 根据项目需要命名，常用结构参考本条目"

### 2.4 Researcher 的发现

**§16 品牌化内容可精简：**

> §16.3 Logo 规范中 `#58a6ff` 品牌色、macOS 窗口点、终端美学——这些是 OpenProfile 的视觉语言，不是通用标准。
> 建议：保留 Logo 规格标准（尺寸、格式、存储路径），视觉风格改为"由 Brain 根据项目定位决定"

**§16.5 Badge 套件：**

> Badge 顺序和 URL 模板是通用的，但 "Built with Copilot" badge 做为默认项不合适——
> 下一个项目可能不用 Copilot（虽然不太可能）。建议改为"可选"。

### 2.5 Code-Reviewer 的发现

**全局扫描结论：质量良好。**

> 1143 行、16 章 + 2 附录，章节编号连续、交叉引用准确。
> 唯一建议：尾部修订记录太长，按 CHANGELOG 管理更合理——手册尾部只保留当前版本号和日期，不堆叠历史。

**通过。** Brain 执行，涉及 5 处修改（见 2.1 表格），改为通用占位符或项目无关表述。

### 决议 D-002：合并冗余内容

**通过。** Dev 执行：
- §4.5 Emoji 表删除"含义"列，只保留 Emoji↔Type 映射
- §15.4 精简，核心步骤引用 §5.2，只保留 API 命令部分

### 决议 D-003：§15.2 补充 Bash 等效命令

**通过。** Dev 执行：在 PowerShell 代码块后补充 bash 等效脚本。

### 决议 D-004：§16 品牌化规范泛化

**通过。**
- §16.3 视觉语言改为"参考风格"，不规定具体配色
- §16.5 "Built with Copilot" badge 由必选改为可选
- §16.6 保留角色分工框架

### 决议 D-005：§9.2 文件夹名改为推荐示例

**通过。** 改为"推荐结构，Brain 可根据项目需要调整命名"。

### 决议 D-006：新增内容——Playbook 自身定制指南

**通过。** 新增 §17，内容：
- Brain 带团队进入新项目时，Playbook 的哪些部分需要定制、哪些直接复用
- 零上下文冷启动协议：当项目没有任何现有文件时，Brain 如何从零建立上下文
- Playbook 版本管理：每个项目可以 fork 自己的 Playbook 副本

### 决议 D-007：手册尾部修订记录精简

**通过。** 尾部只保留当前版本"v3.0.0"标记和日期，历史修订迁入 CHANGELOG。

### 决议 D-008：版本号 — v3.0.0

**全员一致通过。**

理由：
- v1.0.0 = 项目初始化
- v2.0.0 = 完整团队方法论建立
- **v3.0.0 = 团队方法论从项目中解耦，成为可移植的通用框架**

这是 MAJOR 级别的定位转变，不是功能增量。
