# docs 目录导航

本目录按“治理 / 指南 / 策略 / 记录”分层，便于跨项目复用。

## 1) 治理（governance）

- `docs/governance/team-playbook.md`：团队协作总手册
- `docs/governance/PLAYBOOK-CHANGELOG.md`：Playbook 独立变更日志（区别于项目 `CHANGELOG.md`）
- `docs/governance/agent-workflow.md`：Agent 协作流
- `docs/governance/design-decisions.md`：设计决策记录
- `docs/governance/release-sop.md`：版本发布 SOP
- `docs/governance/tooling-scaffold.md`：工具层脚手架说明（settings/hooks/mcp/skills）

## 2) 指南（guides）

- `docs/guides/component-guide.md`：组件使用与参数说明

## 3) 策略（strategy）

- `docs/strategy/open-source-strategy.md`：开源增长策略
- `docs/strategy/brand-content-checklist.md`：内容审查清单
- `docs/strategy/brand-discussions-plan.md`：Discussions 发布计划
- `docs/strategy/brand-release-template.md`：品牌发布模板
- `docs/brand/`：品牌草案与阶段性内容

## 4) 记录（records）

- `docs/meetings/`：会议纪要
- `docs/research/`：调研报告
- `docs/reviews/`：审查报告

---

## 迁移原则

- **项目发布历史** 放在 `CHANGELOG.md`
- **方法论演进历史** 放在 `docs/governance/PLAYBOOK-CHANGELOG.md`
- **用户个人偏好** 放在 `.github/USER.md`（跨项目可直接复用）
